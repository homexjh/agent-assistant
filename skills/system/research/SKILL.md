# Research - 系统性研究方法论

## 概述

本 skill 提供系统化的网络研究方法，确保信息收集全面、多维度、有深度。

**核心原则：不要基于单一搜索结果做结论，至少从 3-5 个不同角度收集信息。**

## 何时使用

**必须使用的情况：**
- 用户说"研究一下 X"
- 用户问"什么是 X"、"解释一下 X"
- 生成内容前需要收集信息（报告、PPT、文章）
- 需要对比多个选项
- 需要最新信息，不能仅靠训练数据

## 4 阶段研究法

### Phase 1: 广度探索（建立地图）

**目标：** 了解全局，识别关键维度

**步骤：**
1. 用 3-5 个不同关键词搜索主题
   - 主关键词
   - 应用场景
   - 相关技术/概念
   - 最新动态
   - 问题/挑战

2. 从搜索结果中识别关键维度：
   - 技术维度：原理、实现、优缺点
   - 市场维度：主流产品、市场份额
   - 案例维度：实际应用、成功案例
   - 趋势维度：发展方向、未来预测
   - 对比维度：与替代方案的比较

**示例：**
```
主题：AI 编程助手
搜索：
- "AI coding assistant 2024"
- "GitHub Copilot alternatives"
- "AI programming tools comparison"
- "AI coding assistant limitations"

识别维度：
- 主流产品（Copilot, Cursor, Codeium）
- 技术原理（大模型、代码补全）
- 应用场景（日常开发、代码审查）
- 优缺点（效率提升、准确性问题）
```

### Phase 2: 深度挖掘（逐个击破）

**目标：** 对每个关键维度深入理解

**步骤：**
1. 针对每个维度，设计 2-3 个深入问题
2. 使用 web_search 搜索这些具体问题
3. 用 web_fetch 获取重要页面的完整内容
4. 记录关键发现（引用来源）

**示例：**
```
维度：主流产品
深入搜索：
- "GitHub Copilot pricing features 2024"
- "Cursor IDE vs Copilot detailed comparison"
- "Codeium free tier limitations"

获取内容：
- 官方网站价格页面
- 技术评测文章
- 用户真实评价
```

### Phase 3: 多角度验证（交叉检查）

**目标：** 确保信息全面、客观、准确

**检查清单：**

| 信息类型 | 检查点 | 搜索建议 |
|---------|-------|---------|
| **事实数据** | 是否有具体数字/统计 | "statistics", "market share", "用户数量" |
| **正反观点** | 是否看到批评/局限性 | "criticism", "limitations", "problems" |
| **多方来源** | 是否来自不同网站 | 至少 3 个不同域名 |
| **时效性** | 信息是否最新 | 优先看 2024-2025 年的内容 |
| **权威性** | 来源是否可靠 | 官方文档、知名媒体、学术论文 |

**示例：**
```
- 正面：效率提升 55%（来源：GitHub 官方报告）
- 反面：代码安全漏洞风险（来源：Security Research Blog）
- 数据：市场份额 Copilot 60%, Cursor 20%（来源：Dev Survey 2024）
```

### Phase 4: 综合检查（质量把关）

**自检清单：**

- [ ] 是否从 3-5 个不同角度搜索过？
- [ ] 是否获取了至少 2 个权威来源的全文？
- [ ] 是否有具体数据支持结论？
- [ ] 是否看到了反面意见或局限性？
- [ ] 信息是否来自 2024 年以后？
- [ ] 是否能回答：这是什么、为什么重要、怎么用、有什么问题？

**如果任一项未勾选，返回相应阶段补充研究。**

## 搜索技巧

### 有效查询模式

```
# ❌ 不好的查询
"AI tools"
"best programming language"

# ✅ 好的查询
"AI coding assistant productivity statistics 2024"
"Python vs Rust performance benchmark web services"
"machine learning framework comparison TensorFlow PyTorch 2024"
```

### 时间限定

- 最新动态：用 "2024", "latest", "recent"
- 历史背景：用 "history", "evolution", "timeline"
- 未来趋势：用 "future", "prediction", "roadmap"

### 获取全文

当搜索结果看起来很有价值时，**一定要用 web_fetch 获取完整内容**，不要只看摘要。

**重点获取：**
- 官方文档/白皮书
- 技术博客深度文章
- 学术论文摘要
- 行业报告

## 输出格式

研究完成后，按以下结构组织发现：

```markdown
## 研究主题：[主题名]

### 1. 核心概念
[用 2-3 句话概括主题]

### 2. 关键发现
#### [维度1]
- 发现 1（来源）
- 发现 2（来源）

#### [维度2]
- ...

### 3. 数据与案例
- 统计数据：...（来源）
- 典型案例：...（来源）

### 4. 优缺点/争议
- 优点：...
- 局限性：...

### 5. 来源汇总
1. [来源名称](URL) - 可靠性评估
2. ...
```

## 常见错误

- ❌ 只做 1-2 次搜索就下结论
- ❌ 只看搜索摘要，不获取全文
- ❌ 只搜正面信息，忽略问题和批评
- ❌ 使用过时信息（2 年以上）
- ❌ 不记录信息来源

## 快速模式（时间紧迫时）

如果用户要求快速回答：

1. 做 2-3 次关键搜索
2. 获取 1-2 个权威来源
3. 明确告知用户这是快速调研，建议深入研究
